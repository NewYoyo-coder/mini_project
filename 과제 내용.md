<개발 요구 사항 - 꼭 들어가야 하는 것>

1. README.md
2. 아래 항목은 반드시 포함
   - 취미 "디렉토리" / 하위에 항목별 md 파일 생성
     ==> 각각의 항목은 이슈로 추가(이슈 만들고 comment 쓸 수 있음)
   - 좋아하는 것 "디렉토리"

   (ex)
   README.md
   hobby/
   취미1.md
   취미2.md
   README.md
   favorite/
   좋아하는 것1.md

3. 커밋 메시지 스타일
   ""#comment 번호""

4. 반드시 사용해야 하는 명령어
   git commit --amend
   git commit --fixup
   git branch
   git checkout
   git checkout -branch
   git merge
   git rebase
   git stash
   git remote, fetch, pull

- 충돌 케이스 생성

1. origin/main의 최신 상태를 받아(fetch) 새로운 브랜치를 로컬에서 생성
2. 1에서 만든 새로운 브랜치에서 ~/README.md 수정
3. 원격에서도 같은 파일 수정 + main 브랜치에 커밋해버리기
4. 로컬에서 fetch -> 로컬의 main브랜치를 origin/main 으로 rebase
5. 로컬의 main 브랜치로 로컬의 변경사항 merge (충돌 발생)
6. 충돌 해결해보기
